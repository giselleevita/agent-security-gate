## Publish checklist

### LinkedIn

1. Open [`linkedin.md`](linkedin.md) and copy the body (omit this README).
2. Paste into a new LinkedIn post.
3. Confirm links resolve: repo, technical brief, full post, portfolio, **v0.6.0 release**.
4. Optional: add a first comment with the canonical URL if LinkedIn strips link preview.

### dev.to

1. Sign in → **Create post** → **Import from URL** is optional; pasting markdown works.
2. Paste contents of [`devto.md`](devto.md) including the YAML front matter.
3. Verify **Canonical URL** in dev.to settings matches the GitHub doc above (front matter pre-fills this).
4. Set cover image optional; publish or schedule.

### After publishing

- Add the live LinkedIn/dev.to URL as a comment on the GitHub blog issue or your tracker (optional).
- Do not change the canonical URL when updating syndicated copies — edit GitHub first, then refresh drafts.
